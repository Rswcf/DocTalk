## Feature 1: PDF/DOCX Export + Shareable Links

### 1.1 Export System

**Goal:** Extend current Markdown-only export to PDF and DOCX formats.

**Backend endpoint:**

```
GET /sessions/{session_id}/export?format=pdf|docx|md
```

- Auth required (session owner only)
- Returns file as streaming response with `Content-Disposition: attachment`
- Gating: Free = Markdown only, Plus+ = PDF/DOCX (matches existing export gate)

**Export content structure:**
1. Header: document/collection name, date, DocTalk branding
2. Message sequence: user questions (bold) + assistant answers
3. Footnotes section: `[1] Page 3, contract.pdf — "The parties agree to..."` format
4. Footer: "Generated by DocTalk — www.doctalk.site"

**Libraries:**
- DOCX: `python-docx` — creates Word document with styled paragraphs + footnotes table
- PDF: `xhtml2pdf` — renders HTML template to PDF (lighter than weasyprint, no system deps)
- Markdown: move existing frontend export logic to backend for consistency

**Backend implementation (`/backend/app/api/export.py`):**

```python
@router.get("/sessions/{session_id}/export")
async def export_session(
    session_id: UUID,
    format: Literal["pdf", "docx", "md"] = Query("md"),
    user: User = Depends(get_current_user),
    db: AsyncSession = Depends(get_db),
):
    session = await verify_session_access(session_id, user, db)
    messages = await get_session_messages(session_id, db)

    if format in ("pdf", "docx"):
        check_subscription(user, min_plan="plus")

    if format == "md":
        content = render_markdown(session, messages)
        return StreamingResponse(content, media_type="text/markdown", ...)
    elif format == "docx":
        buffer = render_docx(session, messages)
        return StreamingResponse(buffer, media_type="application/vnd.openxmlformats-officedocument.wordprocessingml.document", ...)
    elif format == "pdf":
        buffer = render_pdf(session, messages)
        return StreamingResponse(buffer, media_type="application/pdf", ...)
```

**Frontend changes:**
- `ChatPanel.tsx`: Replace single "Export" button with dropdown: Markdown / PDF / DOCX
- Remove `frontend/src/lib/export.ts` client-side export — all export via backend API
- Download triggered via `window.open(proxyUrl)` or fetch + blob download

### 1.2 Shareable Links

**Goal:** Users can generate a read-only link to share a conversation. Recipients see messages + citation snippets but cannot access the original document.

**Database — new table `shared_sessions`:**

```sql
CREATE TABLE shared_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID NOT NULL REFERENCES chat_sessions(id) ON DELETE CASCADE,
    share_token UUID NOT NULL UNIQUE DEFAULT gen_random_uuid(),
    created_by UUID NOT NULL REFERENCES users(id),
    expires_at TIMESTAMPTZ,  -- NULL = never expires
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_shared_sessions_token ON shared_sessions(share_token);
```

**Backend endpoints (`/backend/app/api/sharing.py`):**

```
POST /sessions/{session_id}/share
  → { share_token, url, expires_at }
  Auth required. Creates or returns existing share for this session.

DELETE /sessions/{session_id}/share
  → 204
  Auth required. Revokes share link.

GET /shared/{share_token}
  → { session_title, messages[], document_name, created_at }
  No auth required (public).
  Messages include: role, content, citations (textSnippet + page + documentFilename only).
  Excludes: bboxes, documentId, chunkId, confidenceScore.
```

**Frontend — new page `/shared/[token]/page.tsx`:**
- Server component fetches shared session data via backend API
- Renders read-only conversation: message bubbles + simplified citation cards (snippet + page + doc name)
- No PDF viewer, no interactive chat
- Bottom CTA banner: "Powered by DocTalk — Try it free" → link to homepage
- Mobile-responsive single-column layout
- OG meta tags for link previews: title = session title, description = first assistant message preview

**Frontend — share button in ChatPanel:**
- New "Share" icon button in ChatPanel header (next to export)
- Click → calls POST /share → copies URL to clipboard → toast notification "Link copied!"
- If already shared: shows "Copy link" + "Revoke" options
- Gating: Free users get 3 active shares, Plus+ unlimited

## Feature 2: Citation Confidence Indicators

### 2.1 Backend Changes

**Retrieval service — pass scores through:**

Current `search()` and `search_multi()` return chunk dicts. Add `score` field from Qdrant results.

```python
# retrieval_service.py — in search() and search_multi()
results = [
    {
        "chunk_id": str(point.id),
        "text": point.payload["text"],
        "document_id": point.payload.get("document_id"),
        "score": point.score,  # NEW: Qdrant cosine similarity
        # ... existing fields
    }
    for point in qdrant_results
]
```

**Chat service — include in citations:**

In `RefParserFSM` output / citation construction, add:

```python
citation = {
    # ... existing fields
    "confidence_score": round(chunk["score"], 3),  # NEW: 0.0-1.0
    "context_text": chunk["text"][:300],  # NEW: extended context for hover
}
```

**SSE citation event — include new fields:**

```json
{"type": "citation", "data": {
    "refIndex": 1,
    "chunkId": "...",
    "page": 3,
    "bboxes": [...],
    "textSnippet": "...",
    "confidenceScore": 0.87,
    "contextText": "Extended context up to 300 chars...",
    "documentId": "...",
    "documentFilename": "contract.pdf"
}}
```

### 2.2 Frontend Changes

**Types update (`/frontend/src/types/index.ts`):**

```typescript
interface Citation {
    // ... existing fields
    confidenceScore?: number;  // NEW
    contextText?: string;      // NEW
}
```

**CitationCard hover popover:**

When user hovers over inline `[1]` marker in a message:
- Show popover/tooltip with:
  - Confidence bar: colored (green >0.8, yellow 0.5-0.8, red <0.5) + percentage
  - Source: document filename + page number
  - Context: `contextText` (~300 chars of surrounding text)
  - Section: `sectionTitle` if available
- Popover appears after 300ms hover delay, stays open while mouse is over it
- On mobile: tap to toggle popover

**No changes to:** existing citation click behavior (still navigates to page in single-doc view), message rendering, chat flow.

### 2.3 Shared sessions exclusion

Confidence scores and context text are NOT included in shared session responses (GET /shared/{token}). Shared views show simplified citation cards only (snippet + page + doc name).

The following already work:
- `POST /collections/{id}/sessions` — creates collection-scoped session

## Feature 4: Legal / Academic Mode (Overlay)

### 4.1 Backend

**Database migration — add domain_mode to sessions:**

```sql
ALTER TABLE chat_sessions ADD COLUMN domain_mode VARCHAR(20) DEFAULT NULL;
-- Valid values: NULL (default), 'legal', 'academic'
```

**ChatRequest schema update:**

```python
class ChatRequest(BaseModel):
    message: str
    mode: Literal["quick", "balanced", "thorough"] = "balanced"
    domain_mode: Optional[Literal["legal", "academic"]] = None  # NEW
    locale: str = "en"
```

**Domain rules (`/backend/app/core/model_profiles.py`):**

```python
DOMAIN_RULES: dict[str, list[str]] = {
    "legal": [
        "Every factual claim MUST have a citation [n] to a specific document fragment. If the document does not contain relevant information, state: 'The document does not contain information on this topic.'",
        "Never generate, infer, or paraphrase legal conclusions not directly supported by the document text.",
        "Use precise legal language: 'The document states...' rather than 'According to...' or 'It seems...'",
        "When quoting, preserve the original wording exactly. Mark any omissions with [...].",
        "If multiple interpretations are possible, present each with its supporting citation.",
    ],
    "academic": [
        "Every claim MUST cite the specific document fragment with [n]. Clearly distinguish direct quotes from paraphrased content.",
        "For comparative questions, cite multiple sources and note agreements or contradictions between them.",
        "Include section titles and page numbers in your analysis when referencing specific parts of a document.",
        "Use academic register: hedging language for uncertain claims ('the data suggests...'), definitive language only for directly quoted content.",
        "If the documents do not contain sufficient evidence to answer, explicitly state the limitation rather than speculating.",
    ],
}
```

**Chat service integration (`chat_service.py`):**

In system prompt construction, after the existing Rules section:

```python
if domain_mode and domain_mode in DOMAIN_RULES:
    rules_text += f"\n\n## {domain_mode.title()} Mode Rules\n"
    for i, rule in enumerate(DOMAIN_RULES[domain_mode], start=len(base_rules) + 1):
        rules_text += f"{i}. {rule}\n"
```

**Session persistence:**

When `domain_mode` is sent in ChatRequest:
1. Update `session.domain_mode` in DB (so subsequent messages inherit it)
2. Use the stored `domain_mode` if not provided in request (sticky per session)

**Citation type classification (Legal mode only):**

In legal mode, add post-processing to classify citations:

```python
if domain_mode == "legal":
    for citation in citations:
        chunk_text = citation["text_snippet"]
        response_text = extract_surrounding_text(response, citation["offset"], window=100)
        # Simple heuristic: if response quotes chunk verbatim (>80% overlap), it's "direct"
        citation["citation_type"] = "direct_quote" if text_overlap(chunk_text, response_text) > 0.8 else "inference"
```

### 4.2 Frontend

**Domain mode toggle in ChatPanel:**

Below the existing Quick/Balanced/Thorough selector, add a secondary row:

```
[Quick] [Balanced] [Thorough]
[Default] [Legal] [Academic]
```

- Small pill buttons, muted style (zinc-600 border, zinc-900 bg when active)
- Default = no domain mode (current behavior)
- Legal = amber accent when active
- Academic = blue accent when active
- Selection persists per session (stored in session, sent in ChatRequest)
- Tooltip on each: "Legal: Strict citation mode for legal documents" / "Academic: Research-grade citations with source attribution"

**Legal mode citation enhancements:**
- Citation cards show `citation_type` badge: "Direct Quote" (green) / "Inference" (yellow)
- This helps lawyers distinguish between verbatim document text and AI interpretation

**Academic mode citation enhancements:**
- Citation cards show `sectionTitle` prominently (already available in chunk data)
- Cards show page number in "p. 42" academic format

**No changes to:** credit system, model selection, billing. Domain mode is a prompt-layer overlay only.

## Cross-Cutting Concerns

### API Proxy
All new backend endpoints go through the existing Next.js proxy at `/api/proxy/[...path]`. No proxy changes needed.

### i18n
- New UI strings (button labels, tooltips, toast messages) added to all 11 locale files
- Export content remains in the document's language (not translated)

### Testing
Each feature includes:
- Backend: pytest unit tests for new endpoints + service logic
- Frontend: manual testing checklist (no existing test framework to extend)
- Smoke test: end-to-end flow verification

### Deployment
Each feature is a separate PR → merge to main → test → merge to stable → deploy:
- Vercel auto-deploys frontend on stable push
- Railway: `railway up --detach` for backend
- DB migrations: `alembic upgrade head` before backend deploy
